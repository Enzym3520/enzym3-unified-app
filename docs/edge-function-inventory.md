## Before you delete anything
For each cleanup candidate, confirm it is NOT triggered by:
1. **n8n** — search your workflows for the function name / `…/functions/v1/<name>`.
2. **Supabase scheduled functions / pg_cron** — Dashboard → Database → Cron Jobs,
   and Edge Functions → schedules.
3. **Third-party webhooks** — Stripe/JaaS/Spotify dashboards pointing at the URL.

A function with zero code refs AND no n8n/cron/webhook trigger is safe to remove.
Delete in small batches, deploy, and watch logs for 404s on function invokes.

## Recommendation
1. Resolve the obvious duplicates first (e.g. pick one of send-welcome-email vs
   send-signup-confirmation; one DJ-vs-vendor invite path) — these are confusing,
   not just unused.
2. Verify the couple/partner `validate-*` group against n8n before touching — it's
   the riskiest to remove.
3. Leave cron candidates until you've confirmed their schedules in one pass.
